# Android 高通 (Adreno) 啟用 Lumen + Vulkan Desktop (SM5) 配置指南

> 適用版本：Unreal Engine 5.7.4-release  
> 日期：2026-05-19  
> 所有資訊均從 UE 5.7.4 source 中驗證

---

## 一、核心架構認知

在 UE 5.7.4 中，**Lumen 在 Android 上只能跑在 `SP_VULKAN_SM5_ANDROID` shader platform** — 也就是俗稱的「Vulkan Desktop」模式。它不是 Mobile Renderer 的延伸，而是一個獨立的 SM5 路徑。

關鍵原始碼判斷 (`Engine/Source/Runtime/RenderCore/Private/RenderUtils.cpp:2358-2383`)：
```cpp
bool DoesPlatformSupportLumenGI(EShaderPlatform Platform, bool bSkipProjectCheck)
{
    const bool bVulkanSM5 = Platform == SP_VULKAN_SM5 || Platform == SP_VULKAN_SM5_ANDROID || ...
    return (bSkipProjectCheck || bLumenSupported)
        && FDataDrivenShaderPlatformInfo::GetSupportsLumenGI(Platform)   // (1) 平台旗標
        && (IsFeatureLevelSupported(Platform, ERHIFeatureLevel::SM6) || bVulkanSM5 || ...)  // (2) FL ≥ SM5
        && !IsForwardShadingEnabled(Platform);                            // (3) 禁用 Forward
}
```

`Engine/Config/Android/DataDrivenPlatformInfo.ini` 中已將 `VULKAN_SM5_ANDROID` 設為支援 Lumen：
```ini
[ShaderPlatform VULKAN_SM5_ANDROID]
MaxFeatureLevel=SM5
bSupportsLumenGI=true
bSupportsDistanceFields=true
bSupportsGPUScene=true
bSupportsRayTracing=true
bSupportsInlineRayTracing=true
bSupportsRayTracingShaders=false   ; ← 注意：僅支援 Inline RT，沒有 RayGen shader
```

## 二、專案配置 — `Config/DefaultEngine.ini`

```ini
[/Script/Engine.RendererSettings]
; ---- 全域必要前置 ----
r.Mobile.ShadingPath=1                          ; Deferred (Lumen 必須)
r.AllowStaticLighting=False                     ; Lumen 與 Static Lighting 互斥
r.ForwardShading=False
r.GenerateMeshDistanceFields=True               ; Lumen Software RT 必須
r.DistanceFieldBuild.EightBit=True              ; 降低 DF 記憶體 (建議行動端)

; ---- Lumen 啟用 ----
r.DynamicGlobalIlluminationMethod=1             ; 1 = Lumen
r.ReflectionMethod=1                            ; 1 = Lumen
r.Lumen.Supported=1                             ; ReadOnly，需在 cook 前設定
r.Lumen.TraceMeshSDFs=1                         ; Software RT 主要路徑
r.Lumen.HardwareRayTracing=0                    ; Adreno HW RT 不穩，強制軟體 RT

; ---- Mobile 平台共用旗標 ----
r.Mobile.SupportGPUScene=1                      ; Lumen 需要 GPU Scene
r.Mobile.AllowDeferredShadingOpenGL=0           ; 避免 OpenGL 走錯路徑
r.MobileHDR=True                                ; Deferred 強制要求

[/Script/AndroidRuntimeSettings.AndroidRuntimeSettings]
bSupportsVulkan=True
bSupportsVulkanSM5=True                         ; ★ 開啟 "Vulkan Desktop [Experimental]"
bBuildForES31=False                             ; 不需要 ES3.1，可省 APK 大小
bBuildForArm64=True
MinSDKVersion=29                                ; Android 10+ (Device Profile 匹配條件)
TargetSDKVersion=33
```

> ⚠️ `bSupportsVulkanSM5` 對應的 UI 是 **Project Settings → Platforms → Android → Build → "Support Vulkan Desktop [Experimental]"**。它在原始碼裡標註為 *Experimental*。

## 三、Device Profile — `Config/DefaultDeviceProfiles.ini`

UE 5.7.4 內建 `BaseDeviceProfiles.ini:861` 的 GPU 匹配規則：
```ini
; 自動匹配 Adreno 7xx/8xx + Android 10+ + SM5 編譯可用
+MatchProfile=(Profile="Android_Adreno_Vulkan_SM5",Match=(
    (SourceType=SRC_GpuFamily, MatchString="Adreno \(TM\) [7-8][0-9][0-9]"),
    (SourceType=SRC_AndroidVersion, ..., MatchString="10"+),
    (SourceType=SRC_SM5Available, MatchString="true")))
```

**只有 Adreno 7xx (Snapdragon 8 Gen 1+) 與 Adreno 8xx (Snapdragon 8 Gen 3 / 8 Elite) 在 Android 10 以上會自動切到 SM5 path**。

在專案層級覆寫此 profile 加上 Lumen 微調：
```ini
[Android_Adreno_Vulkan_SM5 DeviceProfile]
DeviceType=Android
BaseProfileName=Android_Vulkan_SM5

; ---- Lumen scalability (Mobile-tuned) ----
+CVars=r.Lumen.DiffuseIndirect.Allow=1
+CVars=r.Lumen.ScreenProbeGather.RadianceCache=1
+CVars=r.Lumen.ScreenProbeGather.DownsampleFactor=32      ; 加大降採樣 (預設16)
+CVars=r.Lumen.ScreenProbeGather.TracingOctahedronResolution=8
+CVars=r.Lumen.Reflections.DownsampleFactor=2
+CVars=r.Lumen.Reflections.MaxRoughnessToTraceForFoliage=0.2
+CVars=r.Lumen.TranslucencyVolume.GridPixelSize=64

; ---- Software Ray Tracing 設定 ----
+CVars=r.Lumen.HardwareRayTracing=0                       ; Adreno 強制 SW RT
+CVars=r.Lumen.TraceMeshSDFs.TraceDistance=2000
+CVars=r.Lumen.SceneHeightfieldFootprint=4096

; ---- Surface Cache 減量 ----
+CVars=r.LumenScene.SurfaceCache.AtlasSize=2048           ; 預設 4096，減半
+CVars=r.LumenScene.SurfaceCache.MeshCardsMergedResolutionScale=0.25
+CVars=r.LumenScene.Radiosity.HemisphereProbeResolution=2

; ---- Async Compute (Adreno 7xx+ 支援) ----
+CVars=r.Lumen.AsyncCompute=1

; ---- 其他效能 ----
+CVars=r.Shadow.Virtual.Enable=1                          ; VSM 與 Lumen 配合
+CVars=r.AntiAliasingMethod=4                             ; TSR
+CVars=r.TSR.ShadingRejection.SampleCount=4

; ---- Adreno 驅動限制 workaround ----
+CVars=r.Vulkan.RayTracing.AllowCompaction=0
+CVars=r.Vulkan.UseRealUBs=1
```

## 四、Cook / Package 命令

```bat
Engine\Build\BatchFiles\RunUAT.bat BuildCookRun ^
  -project="%CD%\MyGame.uproject" ^
  -platform=Android ^
  -cookflavor=ASTC ^
  -targetplatform=Android_ASTC ^
  -clientconfig=Development ^
  -build -cook -stage -package -pak ^
  -cmdline="-vulkan -dx12fl12_2 -featurelevelsm5" ^
  -compressed
```

關鍵點：
- `-cookflavor=ASTC` — Adreno 對 ASTC 壓縮支援最完整
- cook 過程中 UBT 偵測 `bSupportsVulkanSM5=true` 後會編譯 `SF_VULKAN_SM5_ANDROID` shader（見 `AndroidTargetPlatformSettings.cpp:176`）
- APK 體積會明顯變大（多一份 SM5 shader）

## 五、執行時驗證 (logcat)

裝置啟動後在 logcat 找：
```
LogAndroid: Vulkan SM5 RHI will be used!                ← 成功進入 Desktop 模式
LogShaderCompilers: ...SF_VULKAN_SM5_ANDROID...         ← shader 已 cook
LogRenderer: Lumen GI enabled                            ← Lumen 啟動
```

失敗訊號：
```
** Vulkan SM5 support is not available
   → 驅動/編譯/cmdline 任一缺失，回退到 ES3.1 Mobile
Lumen is enabled but cannot render, because the project has Lumen disabled in an ini (r.Lumen.Supported = 0)
   → r.Lumen.Supported 在 cook 階段被讀取，必須是 DefaultEngine.ini 中設定
```

## 六、硬體與已知限制

| 項目 | 狀態 |
|---|---|
| Adreno 6xx (SD865 以下) | ❌ 不支援 SM5 path |
| Adreno 7xx (SD8 Gen1/2) | ✅ Lumen GI/Reflections (SW RT) 可運行 |
| Adreno 8xx (SD8 Gen3/Elite) | ✅ 推薦，可開啟 Async Compute |
| Mali GPU | ⚠️ 走 `Android_Mali_Vulkan_SM5`，須額外調 `r.RHICmd.MaxAccelerationStructureBuildScratchSize=0` |
| Hardware Ray Tracing | ❌ Adreno 驅動的 RayGen Shader 支援有限 (`bSupportsRayTracingShaders=false`)，只能用 Inline RT，所以 Lumen 走 SW RT |
| Nanite | ❌ 與 Android Vulkan SM5 互斥 (`bSupportsNanite` 未啟用)，無法同時開 |
| Virtual Shadow Maps | ✅ 支援，建議開啟 |
| Mobile Forward | ❌ 切到 Lumen 後必須使用 Mobile Deferred |

## 七、最小驗證清單

```
[ ] Project Settings → Rendering → Dynamic GI = Lumen, Reflection = Lumen
[ ] Project Settings → Rendering → Forward Shading = Off, Static Lighting = Off
[ ] Project Settings → Rendering → Generate Mesh Distance Fields = On
[ ] Project Settings → Rendering → Mobile → Shading Path = Deferred, Mobile HDR = On
[ ] Project Settings → Android → Support Vulkan = On
[ ] Project Settings → Android → Support Vulkan Desktop [Experimental] = On
[ ] Minimum SDK = 29 (Android 10)
[ ] 測試裝置 GPU 為 Adreno 7xx 以上
[ ] logcat 出現 "Vulkan SM5 RHI will be used!"
```

如果你已有專案，可以先用 `r.DumpShaderPlatformsSupported` 與 `r.ShaderPlatform.Info` 在 PIE/裝置上 dump 確認 `VULKAN_SM5_ANDROID` 已被列入 cook target。
