# 移動端 Lumen + 4-RT GBuffer BasePass 改造方案

> 適用版本：Unreal Engine 5.7.4-release  
> 日期：2026-05-19  
> 目標：在保持 Lumen 啟用的前提下，將 Deferred BasePass 從 6 RT 精簡到 4 RT，適配 Android Vulkan SM5 (Desktop) 行動端管線  
> 前置文件：[2026_05_19_00.md](./2026_05_19_00.md)（Android Lumen 啟用基礎）

---

## 一、前置：UE5.7 預設 6-RT Deferred GBuffer Layout

從 `Engine/Source/Runtime/RenderCore/Private/GBufferInfo.cpp:362-412` 解析，預設 Layout (Lumen 啟用、`bHasVelocity=true`、`bHasPrecShadowFactor=true`)：

| Slot | RT Name | Format | 用途 | Lumen 依賴 |
|---|---|---|---|---|
| 0 | `Lighting` | 11.11.10 | SceneColor (Emissive + 間接照明累加) | ✅ 寫入 |
| 1 | `GBufferA` | 10.10.10.2 | WorldNormal + PerObjectGBufferData | ✅ 採樣 |
| 2 | `GBufferB` | 8.8.8.8 | Metallic / Specular / Roughness / ShadingModelID | ✅ 採樣 |
| 3 | `GBufferC` | 8.8.8.8 (sRGB) | BaseColor / GBufferAO / IndirectIrradiance | ✅ 採樣 |
| 4 | `Velocity` | 16.16 / 16.16.16.16 | Per-pixel velocity | ⚠️ 間接 (TSR/Reprojection) |
| 5 | `GBufferD` | 8.8.8.8 | CustomData (per shading model) | ❌ 僅 DefaultLit 時不用 |
| 6 | `GBufferE` | 8.8.8.8 | Precomputed Shadow Factors (Static Lighting) | ❌ Lumen-only 場景不用 |

### 我們要的 4-RT 目標 Layout

| Slot | RT Name | 說明 |
|---|---|---|
| 0 | `Lighting` (SceneColor) | 保持 |
| 1 | `GBufferA` (Normal) | 保持 |
| 2 | `GBufferB` (M/S/R + ShadingModelID) | 保持 |
| 3 | `GBufferC` (BaseColor + AO) | 保持 |

→ **省下 GBufferD/E/Velocity**：節省約 **12 bytes/pixel** 頻寬，對 Tile-Based Renderer (Adreno/Mali) 收益巨大。

## 二、技術可行性分析：為什麼可以省掉這三張 RT

### 2.1 GBufferE — Precomputed Shadow Factor

`Engine/Source/Runtime/Engine/Private/ShaderCompiler/ShaderGenerationUtil.cpp:2300`：
```cpp
Ret.bHasPrecShadowFactor = IsStaticLightingAllowed();
```
**已在 [Lumen 配置](./2026_05_19_00.md) 設定 `r.AllowStaticLighting=False` → GBufferE 不會被分配。**

採樣端有完整保護 (`Engine/Shaders/Private/DeferredShadingCommon.ush:1227`)：
```hlsl
#if ALLOW_STATIC_LIGHTING
    float4 GBufferE = Texture2DSampleLevel(SceneTexturesStruct.GBufferETexture, ...);
#else
    float4 GBufferE = 1;
#endif
```
**無須額外改動。**

### 2.2 Velocity — BasePass Velocity

`Engine/Source/Runtime/Engine/Private/ShaderCompiler/ShaderGenerationUtil.cpp:2302`：
```cpp
Ret.bHasVelocity = (IsUsingBasePassVelocity(Platform) || Layout == GBL_ForceVelocity) ? 1 : 0;
```
透過設定 `r.VelocityOutputPass=0`（讓 Velocity 改在 Depth Prepass 輸出）或 `r.BasePassWriteVelocity=0`，可移出 BasePass。

> Velocity 仍會生成於獨立 `SceneVelocity` 紋理 (`SceneTextures.cpp:703`)，不影響 Lumen TSR / Temporal Reprojection。

### 2.3 GBufferD — CustomData（**關鍵改造點**）

GBufferD 由「shading model 需求」自動觸發。`Engine/Shaders/Private/BasePassCommon.ush:44`：
```hlsl
#define WRITES_CUSTOMDATA_TO_GBUFFER (USES_GBUFFER && (
    MATERIAL_SHADINGMODEL_SUBSURFACE ||
    MATERIAL_SHADINGMODEL_PREINTEGRATED_SKIN ||
    MATERIAL_SHADINGMODEL_SUBSURFACE_PROFILE ||
    MATERIAL_SHADINGMODEL_CLEAR_COAT ||
    MATERIAL_SHADINGMODEL_TWOSIDED_FOLIAGE ||
    MATERIAL_SHADINGMODEL_HAIR ||
    MATERIAL_SHADINGMODEL_CLOTH ||
    MATERIAL_SHADINGMODEL_EYE))
```

問題：**`FetchFullGBufferInfo` 無條件分配 GBufferD slot**，因為 layout 是專案層級而非材質層級。即便場景中所有材質都是 DefaultLit，GBufferD 紋理仍佔用 VRAM 與頻寬。

`Engine/Source/Runtime/RenderCore/Private/GBufferInfo.cpp:370-380`：
```cpp
if (Params.bHasVelocity == 0 && Params.bHasTangent == 0)
{
    TargetGBufferD = 4;
    Info.Targets[4].Init(GBT_Unorm_8_8_8_8, TEXT("GBufferD"), false, true, true, true); // ← 無條件
    if (Params.bHasPrecShadowFactor) { ... }
}
```

→ **需新增 `FGBufferParams::bHasCustomData` 旗標 + 條件編譯**。

### 2.4 Lumen 對 GBufferD 的真實依賴

`Engine/Shaders/Private/Lumen/LumenMaterial.ush:619-657`，`InternalReadMaterialData_GBuffer` 從 `FGBufferData` 結構讀取：
```hlsl
Out.Anisotropy = GBufferData.Anisotropy;             // ← 來自 GBufferD
Out.TopLayerRoughness = GetClearCoatRoughness(...);  // ← 來自 GBufferD
Out.GBufferData = GBufferData;                       // ← 含 CustomData
```

**但 `FGBufferData` 結構本身在 C++ 端是抽象的**：如果 GBufferD 紋理不存在，採樣會經由 `Engine/Source/Runtime/Renderer/Private/SceneTextures.cpp:1077`：
```cpp
SceneTextureParameters.GBufferDTexture = SystemTextures.Black; // 預設黑色 1x1 dummy
```

→ **Anisotropy=0、ClearCoat=0、CustomData=0**。對 DefaultLit 材質完全無影響；非 DefaultLit 材質會 fallback 成「無 anisotropy、無 clearcoat 的 DefaultLit」。

**這正是我們要的行為**：行動端 Lumen 場景明確只支援 DefaultLit + Unlit。

## 三、改造方案：8 處檔案修改

### 改動點 1：FGBufferParams 新增 `bHasCustomData` 旗標

**檔案**：`Engine/Source/Runtime/RenderCore/Public/GBufferInfo.h:293`

```cpp
struct FGBufferParams
{
    EShaderPlatform ShaderPlatform = SP_NumPlatforms;
    int32 LegacyFormatIndex = 0;
    bool bHasVelocity = false;
    bool bHasTangent = false;
    bool bHasPrecShadowFactor = false;
    bool bUsesVelocityDepth = false;
    bool bHasSingleLayerWaterSeparatedMainLight = false;
    bool bHasCustomData = true;   // ★ 新增，預設 true 保持相容

    bool operator == (const FGBufferParams& RHS) const
    {
        return ShaderPlatform == RHS.ShaderPlatform
            && LegacyFormatIndex == RHS.LegacyFormatIndex
            && bHasVelocity == RHS.bHasVelocity
            && bHasTangent == RHS.bHasTangent
            && bHasPrecShadowFactor == RHS.bHasPrecShadowFactor
            && bUsesVelocityDepth == RHS.bUsesVelocityDepth
            && bHasSingleLayerWaterSeparatedMainLight == RHS.bHasSingleLayerWaterSeparatedMainLight
            && bHasCustomData == RHS.bHasCustomData;   // ★
    }
    // operator != 同理新增比較
};
```

### 改動點 2：`FetchFullGBufferInfo` 條件性跳過 GBufferD

**檔案**：`Engine/Source/Runtime/RenderCore/Private/GBufferInfo.cpp:362-412`

```cpp
// 原始 (僅修改 "no velocity, no tangent" 分支 — 我們的目標路徑)
if (Params.bHasVelocity == 0 && Params.bHasTangent == 0)
{
    if (Params.bHasCustomData)   // ★ 新增條件
    {
        TargetGBufferD = 4;
        Info.Targets[4].Init(GBT_Unorm_8_8_8_8, TEXT("GBufferD"), false, true, true, true);

        if (Params.bHasPrecShadowFactor)
        {
            TargetGBufferE = 5;
            Info.Targets[5].Init(GBT_Unorm_8_8_8_8, TEXT("GBufferE"), false, true, true, true);
        }
    }
    else if (Params.bHasPrecShadowFactor)   // ★ GBufferE 槽位向前位移
```

**SLW SeparatedMainDirLight 修正**（同檔 line 415）：

## 四、RDG 安全性審查

### 4.1 已內建保護機制

✅ **`SetupSceneTextureUniformParameters` (SceneTextures.cpp:1077)**：所有 GBuffer Texture 預設指向 `SystemTextures.Black`，未 produce 時不會 null binding。

✅ **`GetIfProduced` (SceneTextureParameters.cpp:30)**：返回 dummy 黑色紋理。

✅ **`HasBeenProduced` 檢查** (SceneTextures.cpp:1118)：只有真正 produce 才會綁定。

### 4.2 需要人工檢查的 RDG 風險點

⚠️ **BasePass RenderTarget Count 必須與 PSO 一致**

`Engine/Source/Runtime/Renderer/Private/BasePassRendering.cpp:1183`：
```cpp
const FGBufferBindings& GBufferBindings = SceneTextures.Config.GBufferBindings[GBL_Default];
if (!CVarClearGBufferDBeforeBasePass.GetValueOnRenderThread()
    && GBufferBindings.GBufferD.Index > 0
    && GBufferBindings.GBufferD.Index < (int32)BasePassTextureCount)
{
    PassParameters->RenderTargets[GBufferBindings.GBufferD.Index].SetLoadAction(ERenderTargetLoadAction::ENoAction);
}
```
**這段已經有 `Index > 0` 守衛**：當 `bHasCustomData=false`、`GBufferD.Index=-1` 時自動跳過。✅

⚠️ **PSO 緩存命中率**

切換 `r.GBuffer.StripCustomData` 後，PSO Cache、Shader Map、DDC 全部失效（因 DDC Key 變化）。**生產環境必須在打包前敲定設定，避免運行期切換導致 1-2 秒的 shader 重編譯卡頓**。

⚠️ **MRT 索引漂移**

shader 中所有寫死的 `Out.MRT[N]` 必須校驗。`BasePassPixelShader.usf:2373`：
```hlsl
Out.MRT[GBUFFER_HAS_VELOCITY ? 5 : 4] = OutGBufferD;
```
此處 `Refactor` 路徑會由 `PIXELSHADEROUTPUT_MRT*` 自動處理；**但若你的專案有自訂的 legacy BasePass 變體**，需手動 wrap：
```hlsl
#if GBUFFER_HAS_CUSTOMDATA
    Out.MRT[GBUFFER_HAS_VELOCITY ? 5 : 4] = OutGBufferD;
#endif
#if GBUFFER_HAS_PRECSHADOWFACTOR
    Out.MRT[(GBUFFER_HAS_VELOCITY ? 6 : 5) - (GBUFFER_HAS_CUSTOMDATA ? 0 : 1)] = OutGBufferE;
#endif
```

## 五、Lumen 通路驗證 (各 Pass 對 GBuffer 的依賴)

| Lumen Pass | GBufferA | GBufferB | GBufferC | GBufferD | GBufferE |
|---|---|---|---|---|---|
| LumenScreenProbeGather | ✓ Normal | ✓ Rough | ✓ BaseColor | ⚠️ Anisotropy → 0 安全 | ✗ |
| LumenReflections | ✓ Normal | ✓ Rough/Metallic | ✓ | ⚠️ ClearCoatRough → 0 安全 | ✗ |
| LumenSurfaceCache (Card Capture) | ✓ | ✓ | ✓ | ⚠️ 取決於材質 BasePass 寫入 | ✗ |
| LumenRadiosity | ✓ | ✓ | ✓ | ✗ | ✗ |
| LumenSceneLighting | ✓ | ✓ | ✓ | ✗ | ✗ |
| LumenVisualize | ✓ | ✓ | ✓ | ⚠️ ShadingModel 顯示退化 | ✗ |
| TSR / Reprojection | - | - | - | - | - (用 Velocity 紋理) |

**驗證來源**：`grep -r "GBufferDTexture" Engine/Source/Runtime/Renderer/Private/Lumen/` 無命中，Lumen 不直接採樣 GBufferD 紋理。所有 CustomData 讀取均透過 `FGBufferData.CustomData/Anisotropy`，當紋理為 `SystemTextures.Black` 時返回 0，BxDF 行為退化為純 DefaultLit — **這正是我們需要的**。

## 六、項目配置 — `Config/DefaultEngine.ini`

```ini
[/Script/Engine.RendererSettings]
; ---- 既有 Lumen + Android Vulkan SM5 配置 (見 2026_05_19_00.md) ----
r.AllowStaticLighting=False               ; ★ 自動移除 GBufferE
r.DynamicGlobalIlluminationMethod=1
```

## 七、限制與權衡

| 損失 | 影響 | 緩解 |
|---|---|---|
| Subsurface Scattering | 皮膚/植被 SSS 退化為 DefaultLit | 用 emissive + custom shader 模擬，或保留 GBufferD（不執行此改造） |
| Clear Coat | 車漆/亮面退化為單層 specular | 同上 |
| Hair Shading | 頭髮卡片變平板 | 行動端通常使用簡化卡片，可接受 |
| Anisotropy | 拉絲金屬退化為 isotropic | 通常行動端 LOD 也會關 |
| Eye Shading | 眼球退化為 DefaultLit | 行動端罕用 |
| Cloth | 布料 sheen 消失 | 行動端可接受 |

**保留的功能**：
✅ Lumen GI（Diffuse + Specular）  
✅ Virtual Shadow Maps  
✅ DefaultLit + Unlit  
✅ Metallic / Roughness / Specular  
✅ Decals（除非 decal 寫入 CustomData）  
✅ Lumen Reflections  
✅ TSR / Temporal AA

## 九、回退策略

若改造後出現問題，回退只需：
```
r.GBuffer.StripCustomData=0
```
**因為所有改動都在 `bHasCustomData=true` 時走原路徑**，DDC Key 會切換，shader 重編後恢復 6-RT。

不建議直接刪除 `bHasCustomData` 旗標——保留作為可選功能，未來可擴展到其他平台。
