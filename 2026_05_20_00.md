## 二、Mobile Renderer 三類 Subpass 的串接內容

### 2.1 `ESubpassHint::DepthReadSubpass`（Mobile Forward）

Subpass | Color Attachments | Depth | Input Attachments | 內容
---|---|---|---|---
0 | SceneColor (+ optional GBufferA Normal) | Write | — | DepthPrepass、BasePass、Sky、PostRenderBasePass
1 | SceneColor | Read-only | DepthStencil[0] | Decals、Fog、Translucency、Occlusion

→ Mobile Forward 主要靠 **Depth 作為 input attachment** 給 Fog/Decal 使用（深度霧、軟粒子）。

### 2.2 `ESubpassHint::CustomResolveSubpass`（Mobile Forward + MSAA Tonemap）

在 DepthReadSubpass 之後再加一個：

Subpass | Color Attachments | Depth | Input Attachments | 內容
---|---|---|---|---
2 | Resolved BackBuffer | — | SceneColor (MSAA) + Depth | MobileCustomResolve (Tonemap + MSAA Resolve)

→ MSAA Tonemap inline，免去單獨的 resolve pass。

### 2.3 `ESubpassHint::DeferredShadingSubpass`（Mobile Deferred）★

`Engine/Source/Runtime/VulkanRHI/Private/VulkanRenderpass.h:639-720` 建構：

Subpass | Color Attachments | Depth | Input Attachments | 內容
---|---|---|---|---
**0** (main) | SceneColor + GBufferA/B/C/D | Write | — | DepthPrepass + BasePass + Sky
**1** (decal) | SceneColor + GBufferA/B/C/D | Read-only | Depth[0] | `RenderDecals` 修改 GBuffer
**2** (light) | SceneColor only | Read-only | Depth[0] + GBuffer A/B/C/D as [2,3,4,5] | `MobileDeferredShadingPass` 累加 SceneColor、Fog、Translucency

實際串接位於 `Engine/Source/Runtime/Renderer/Private/MobileShadingRenderer.cpp:2363-2391`：
```cpp
RenderMaskedPrePass(RHICmdList, View, ...);                    // subpass 0
RenderMobileBasePass(RHICmdList, View, ...);
PostRenderBasePass(RHICmdList, View);

RHICmdList.NextSubpass();                                       // → subpass 1
RenderDecals(RHICmdList, View, ...);

RHICmdList.NextSubpass();                                       // → subpass 2
MobileDeferredShadingPass(RHICmdList, ..., MobileSSRQuality);
RenderFog(RHICmdList, View);
RenderTranslucency(RHICmdList, View, ...);
```

Shader 透過 `VulkanSubpassFetch1/2/3/4` 讀取 GBuffer (`DeferredShadingCommon.ush:791-828`)：
```hlsl
void MobileFetchGBuffer(in float2 UV, out half4 GBufferA, out half4 GBufferB,
                        out half4 GBufferC, out half4 GBufferD, out float SceneDepth)
{
#if (VULKAN_PROFILE && FRAMEBUFFER_FETCH_ACCESS)
    GBufferA = VulkanSubpassFetch1();
    GBufferB = VulkanSubpassFetch2();
    GBufferC = VulkanSubpassFetch3();
    #if MOBILE_EXTENDED_GBUFFER
        GBufferD = VulkanSubpassFetch4();
    #endif
#else
    GBufferA = Texture2DSampleLevel(MobileSceneTextures.GBufferATexture, ..., UV, 0);
    // 走 SRV 採樣 fallback
#endif
}
```

### 3.3 真正的工作量 — Renderer 架構重構

**Mobile Renderer 把所有相關 pass 包在一個 `GraphBuilder.AddPass` 裡**：
```cpp
GraphBuilder.AddPass(RDG_EVENT_NAME("SceneColorRendering"),
    PassParameters,
    ERDGPassFlags::Raster | ERDGPassFlags::NeverMerge,
    [](FRHICommandList& RHICmdList) {
        RenderBasePass(...);
        RHICmdList.NextSubpass();
        RenderDecals(...);
        RHICmdList.NextSubpass();
        MobileDeferredShadingPass(...);
    });
```

**Deferred (SM5) Renderer 每個階段是獨立的 `AddPass`**：
```cpp
RenderBasePass(GraphBuilder, ...);                  // RDG pass 1，獨立 renderpass
AddResolveSceneDepthPass(GraphBuilder, ...);
RenderLumenSceneLighting(GraphBuilder, ...);        // RDG pass N
RenderLights(GraphBuilder, ...);                    // RDG pass N+1
```

要套上 subpass 必須建立一個**「合併 renderpass」**：把 BasePass、Decals、Direct Lighting 三者改寫成 lambda 內順序執行 + `NextSubpass()`。

## 四、設計方案：`SP_VULKAN_SM5_ANDROID` 子集 Subpass 鏈

### 4.1 推薦的 Subpass 拓樸（針對 Lumen-only 行動端）

延續 [02 文件](./2026_05_19_02.md) 的 3-RT GBuffer 設計：

Subpass | Color Attachments | Depth | Input Attachments | 內容 | Tile cost
---|---|---|---|---|---
**0 BasePass** | GBufferA/B/C | Write | — | DepthPrepass(若 inline) + BasePass | 16 B/px (3×4 + D4)
**1 Decals** | GBufferA/B/C | Read | Depth[0] | Mesh Decals 改寫 GBuffer | 同上
**2 DirectLight** | SceneColor (load) | Read | Depth[0] + GBuffer[2,3,4] | Direct Lighting Accumulate | 20 B/px (3×4 + SC4 + D4)

→ Lumen Indirect 不放進來，因為它需要 GBuffer 作為 **SRV** 跨 pixel 採樣（screen probe scatter/gather），無法用 input attachment（只能讀「同一像素」）。

完整 frame timeline：
```
[Renderpass A — Tile-Local]
  Subpass 0: Depth + BasePass (write GBuffer)
  Subpass 1: Decals (FBF read depth, write GBuffer)
  Subpass 2: Direct Lighting (FBF read GBuffer + depth, write SceneColor)
[Store GBuffer + SceneColor to DRAM]

[Renderpass B — Independent]
  Lumen Scene Update (SW RT)
  Lumen Diffuse Indirect → SceneColor (SRV sampling)
  Lumen Reflections → SceneColor (SRV sampling)

[Renderpass C — Tile-Local]
  Subpass 0: Fog (FBF read depth)
  Subpass 1: Translucency
[Store SceneColor]

[Renderpass D — Post Processing]
```

### 4.4 新增 Combined Renderpass：`RenderBasePassAndLightingWithSubpasses`

**檔案**：`Engine/Source/Runtime/Renderer/Private/DeferredShadingRenderer.cpp` 在 `RenderBasePass` 旁邊新增。

```cpp
void FDeferredShadingSceneRenderer::RenderBasePassAndLightingWithSubpasses(
    FRDGBuilder& GraphBuilder,
    FSceneTextures& SceneTextures,
    const FDBufferTextures& DBufferTextures,
    FInstanceCullingManager& InstanceCullingManager,
    const FSortedLightSetSceneInfo& SortedLightSet)
{
    auto* PassParameters = GraphBuilder.AllocParameters<FSceneTexturesPassParameters>();

    // ★ 同時綁定 SceneColor + GBuffer A/B/C，標記 SubpassHint
    TStaticArray<FTextureRenderTargetBinding, MaxSimultaneousRenderTargets> Bindings;
    uint32 NumRT = SceneTextures.GetGBufferRenderTargets(Bindings);
    PassParameters->RenderTargets = GetRenderTargetBindings(ERenderTargetLoadAction::EClear,
                                                            MakeArrayView(Bindings.GetData(), NumRT));
    PassParameters->RenderTargets.DepthStencil = FDepthStencilBinding(
        SceneTextures.Depth.Target,
        ERenderTargetLoadAction::EClear, ERenderTargetLoadAction::EClear,
        FExclusiveDepthStencil::DepthWrite_StencilWrite);
    PassParameters->RenderTargets.SubpassHint = ESubpassHint::DeferredShadingSubpass;   // ★

    GraphBuilder.AddPass(
        RDG_EVENT_NAME("BasePass+Decals+DirectLight (Tile-Local)"),
        PassParameters,
        ERDGPassFlags::Raster | ERDGPassFlags::NeverMerge,
        [this, &SceneTextures, &SortedLightSet, ...](FRHICommandList& RHICmdList)
        {
            // === Subpass 0: BasePass ===
            for (FViewInfo& View : Views)
            {
                if (!bIsFullDepthPrepassEnabled)
                    RenderPrePassView(RHICmdList, View);   // Masked depth prepass inline
                RenderBasePassView(RHICmdList, View);
            }

            // === Subpass 1: Decals ===
            RHICmdList.NextSubpass();
            for (FViewInfo& View : Views)
            {
                RenderMeshDecals(RHICmdList, View);
            }

            // === Subpass 2: Direct Lighting ===
            RHICmdList.NextSubpass();
            for (FViewInfo& View : Views)
            {
                // 從 input attachment 讀 GBuffer，寫 SceneColor
                RenderDirectLightingWithSubpassFetch(RHICmdList, View, SortedLightSet);
            }
        });
}
```

### 4.6 RDG 約束與屏障處理

關鍵注意點：

1. **Subpass 內不能 RDG SRV 採樣 GBuffer**  
   Direct Lighting Shader 不能再有 `SceneTextureUniformParameters.GBufferATexture` 的 SRV binding，否則 RDG 會建立 Texture Read 依賴，與 Color Attachment Write 衝突。必須改為純 Subpass Fetch。

2. **Renderpass 中途不能 Compute**  
   Lumen Indirect 是 Compute pass，**不能塞進 subpass chain**。它必須在 renderpass 結束、Color Store 之後執行。

3. **Depth Resolve / Store**  
   subpass 鏈結束時必須把 Depth Store 出來（Lumen 需要）。`bResolveDepth` 已在 VulkanRenderpass.h:508 處理。

4. **PSO 必須帶 `SubpassIndex`**  
   `MeshDrawCommand` 自動處理（`GraphicsPSOInit.SubpassIndex = 2` for direct light shaders）。

## 五、收益分析（按 Pass 拆解）

### 5.1 帶寬節省（1080p、3-RT GBuffer + D + SceneColor）

每個獨立 renderpass 在 Adreno 上的成本：
- **Load**: 14-16 B/px (3 RT × 4B + Depth 4B = 16B) ≈ 33 MB
- **Store**: 同上 = 33 MB

當前架構 (`r.VulkanDesktop.UseSubpass=0`)：
```
BasePass renderpass:          Store 16 B/px = 33 MB
  ↓ tile flush
Decals renderpass:    Load 16 B/px + Store 16 B/px = 66 MB
  ↓ tile flush  
DirectLight renderpass: Load 16 B/px + Store 20 B/px = 75 MB
                                                       ─────
                                              小計     174 MB/frame
```

啟用 Subpass (`r.VulkanDesktop.UseSubpass=1`)：
```
Tile-Local renderpass (3 subpasses):  Load 0 + Store 20 B/px = 41 MB
                                                                ─────
                                              小計             41 MB/frame
                                              節省            ~133 MB/frame
```

**60fps 下：節省 ~8 GB/s 記憶體頻寬。** 對 Adreno 7xx 系列（理論頻寬 ~50 GB/s）相當於 16% 系統頻寬。

### 5.2 各部分收益細項

| Pass | 當前 (獨立 RP) | Subpass 鏈 | 收益 |
|---|---|---|---|
| Depth Prepass | 獨立 RP，~6 MB Store | 仍獨立（無法塞，會干擾 Lumen 的 SceneDepth SRV） | 0 |
| **BasePass** | Store 16 B/px | 仍是首個 subpass，但**省下後續 Load** | **-33 MB/frame** |
| **Decals** | Load 16 + Store 16 | Subpass 1，無 Load/Store | **-66 MB/frame** |
| **Direct Light** | Load 16 + Store 4 (SceneColor) | Subpass 2，只 Store SceneColor | **-33 MB/frame** |
| Lumen Indirect | 不變（必須 SRV 採樣） | 不變 | 0 |
| Lumen Reflections | 不變 | 不變 | 0 |
| Fog | Load 4 + Store 4 | 可選 chain 進新 RP | -4 ~ -8 MB |
| Translucency | Load 4 + Store 4 | 可選 chain | -4 MB |
| Tonemap | — | — | 0 |

**主要收益集中在 BasePass→Decals→DirectLight 鏈，占整體 ~75%。**

### 5.3 額外收益：Bin Size / Bin Count

Adreno 把 framebuffer 切成 bin（tile）渲染。bin size 由 tile memory 與 RT 數量決定：

| 場景 | RT 總大小 | Tile Memory | Bin 數量 (1080p) |
|---|---|---|---|
| 當前 (4-RT BasePass) | 16 B/px | 768 KB | ~48 bins |
| 3-RT BasePass (02 文件) | 12 B/px | 768 KB | ~32 bins |
| 3-RT + Subpass (含 SceneColor 同 RP) | 16 B/px | 768 KB | ~48 bins |

Subpass 鏈會在某段時間同時佔用 GBuffer A/B/C + SceneColor + Depth = 5 attachments × 4 B = 20 B/px，使 bin 變回 ~38 個（比純 BasePass 多）。

→ Subpass 內的 **vertex shader 重做次數略增**（bin 數變多），但 **pixel shader 與 ROP 頻寬大幅下降**。整體仍是淨贏，因為現代 GPU 的 pixel/ROP 比 vertex 重得多。

### 5.4 Wallclock 預期

對 Adreno 730 / 740 在 1080p 純 DefaultLit + Lumen 場景：

| 階段 | 當前 (ms) | Subpass (ms) | 改善 |
|---|---|---|---|
| BasePass (3-RT) | 3.5 | 3.5 | 0 |
| Decals | 0.8 | 0.3 | -0.5 ms |
| Direct Light | 2.2 | 1.4 | -0.8 ms |
| Tile flush 隱性成本 | 1.0 | 0.2 | -0.8 ms |
| **總計 BasePass→Light** | **7.5** | **5.4** | **-28%** |
| Lumen Indirect | 6.0 | 6.0 | 0 |
| Frame Total | ~22 ms | ~20 ms | **-9%** |

（數據為估算，依場景 overdraw 與光源密度浮動）

## 六、限制與風險

### 6.1 不能塞進 Subpass 的 Pass

| Pass | 原因 |
|---|---|
| **Lumen Diffuse Indirect** | Compute shader + 需要 GBuffer 跨像素 SRV 採樣 (Screen Probe Gather) |
| **Lumen Reflections** | Compute + Ray Tracing |
| **Lumen Surface Cache Update** | 獨立 Card Capture renderpass |
| **VSM Page Update** | Compute |
| **Shadow Map Render** | 不同 RT (depth-only) |
| **AO (SSAO)** | Compute + 跨像素採樣 |

### 6.2 工程風險

| 風險 | 機率 | 緩解 |
|---|---|---|
| Adreno 驅動 bug 觸發（多 subpass 不常見路徑） | 中 | 灰度 CVar `r.VulkanDesktop.UseSubpass=0/1/2` |
| Tile size 變小導致 bin 數增加，vertex 工作量上升 | 中 | 使用 [02 文件](./2026_05_19_02.md) 的 3-RT 進一步壓縮 |
| PSO permutation 爆炸 (新增 `DESKTOP_SUBPASS_FBF`) | 中 | 僅對 deferred light shader 加 permutation |
| Lumen Direct Lighting 與 subpass 直接光衝突 | 低 | Lumen Direct Lighting 不在 subpass 鏈內，由 Lumen 自己的 pass 處理 |
| 第二個 view（split-screen / VR）破壞 subpass | 中 | 加 `NeverMerge` flag（mobile 已有先例） |
| RDG validation 報「Texture used as both Color and SRV」 | 高 | 必須改寫 Direct Light Shader 為純 SubpassFetch，不能再走 SRV path |

### 6.3 與 [02 文件](./2026_05_19_02.md) 3-RT BasePass 改造的相依關係

如果同時啟用 `r.GBuffer.StripSceneColor=1`（02 文件），那麼：
- **Subpass 0** (BasePass) 不再需要 SceneColor attachment
- **Subpass 2** (DirectLight) 才把 SceneColor 加進 framebuffer

這在 Vulkan 上**不被允許**（VkSubpassDescription 對 attachment 的 lifetime 是 renderpass 級的）。

→ **解法**：SceneColor 從 subpass 0 開始就在 framebuffer 中，但 subpass 0/1 用 `VK_ATTACHMENT_LOAD_OP_DONT_CARE` + `COLOR_WRITE_MASK=0`（CW_NONE），實際不寫；subpass 2 才開啟寫入。Adreno 仍會分配 tile memory，但 0/1 不消耗寫入頻寬。

> 即若採用 [02 文件 方案 B（完全解綁）](./2026_05_19_02.md#四方案-b-詳細改動10-處檔案--2-個新-pass)，Subpass 啟用時需退回 **方案 A (CW_NONE)** 的混合策略。

## 七、最小可行實作（MVP）

階段化交付，每階段獨立可驗證：

### MVP-1：閘門打開 + Shader define 注入（1 週）

1. 新增 `IsDesktopMobileGPUPlatform()` / `CanUseSubpassFramebufferFetch()` 平台檢查
2. 新增 `r.VulkanDesktop.UseSubpass` CVar
3. `MobileAllowFramebufferFetchIniValue` 加白名單
4. `ShaderCompiler.cpp` 加分支注入 `ALLOW_FRAMEBUFFER_FETCH=1` 給 SM5 path

**驗證**：開啟後 shader 編譯通過，無 runtime 改變（hint 還沒接 renderer）。

### MVP-2：BasePass + Decals 兩 Subpass（2 週）

1. 在 Deferred Renderer 包出 `RenderBasePassWithDecalsSubpass()`
2. 不動 Direct Light（仍走獨立 SRV path）
3. `RenderTargets.SubpassHint = DeferredShadingSubpass` 但只走前兩個 subpass
4. RDG validation

**驗證**：BasePass renderpass 內含 BasePass + Decals，tile memory 不 flush。RenderDoc 確認 attachment 連續。

### MVP-3：Direct Light 也進 Subpass（3 週）★ 重頭戲

1. 新增 `DesktopSubpassDirectLightPS` permutation
2. 改寫 GBuffer 取樣為 `VulkanSubpassFetch1/2/3` + `VulkanSubpassDepthFetch`
3. Disable Light Function in subpass mode（會破壞 tile locality）
4. Stencil culling 仍可用（depth/stencil 是 input attachment）

**驗證**：BasePass+Decals+DirectLight 一個 renderpass。RenderDoc 確認 3 subpass。Lumen 仍正常。

### MVP-4：Fog + Translucency 第二 Subpass 鏈（選作）

當 `r.VulkanDesktop.UseSubpass=2` 時，把 Fog + Translucency 包成 DepthReadSubpass 拓樸。

## 八、結論與建議

### 8.1 用戶猜測的證實

**完全證實**：`SP_VULKAN_SM5_ANDROID` 確實**完全不走 subpass 路徑**，原因鏈為：

```
GetSubpassHint
  → IsMobilePlatform(SP_VULKAN_SM5_ANDROID) == false  (MaxFeatureLevel=SM5≠ES3_1)
  → MobileAllowFramebufferFetch(SP_VULKAN_SM5_ANDROID) == false  (不在白名單)
  → ALLOW_FRAMEBUFFER_FETCH shader define = 0  (bUsingMobileRenderer=false)
  → Deferred Renderer 從未呼叫 NextSubpass()  (架構不同)
```

### 8.2 補上 Subpass 是否值得？

| 觀點 | 結論 |
|---|---|
| 技術可行性 | ✅ Vulkan RHI 與 shader intrinsics 已就緒，可移植 |
| 純頻寬節省 | ✅ ~133 MB/frame @ 1080p = ~8 GB/s @ 60fps |
| Wallclock 改善 | ✅ ~9-12% frame time 改善（Adreno 730/740 估算） |
| 工程量 | ⚠️ 中等（4 週 MVP，含 shader permutation） |
| 風險 | ⚠️ 中等（Adreno 驅動 bug + PSO 爆炸） |
| 與 02 文件衝突 | ⚠️ Subpass 要求 SceneColor 全程在 framebuffer，與 02 完全解綁衝突 |

**強烈推薦做 MVP-1 + MVP-2**（閘門打開 + BasePass/Decals subpass），收益約 50% 預估值，風險最低。

**MVP-3（DirectLight FBF）需要看你的場景燈光複雜度**，若使用大量 light functions / IES，subpass 路徑會強制 fallback，得不償失。

### 8.3 與已有改造的整合建議

```ini
[/Script/Engine.RendererSettings]
; ---- Stage 1 (從 00/01 文件)：Lumen on Vulkan SM5 ----
r.AllowStaticLighting=False
r.DynamicGlobalIlluminationMethod=1
r.GBuffer.StripCustomData=1

; ---- Stage 2 (從 02 文件) — 暫時關閉 ----
; r.GBuffer.StripSceneColor=1   ; ★ 與 Subpass 衝突，暫不啟用

; ---- Stage 3 (本文)：補上 Subpass ----
r.VulkanDesktop.UseSubpass=1                  ; BasePass + Decals + DirectLight
r.Mobile.AllowFramebufferFetch=1              ; 同時讓 SM5 path 取得 fetch 權
```

兩階段選擇：
- **Stage 2 (3-RT BasePass)**：節省 ~25% BasePass write 頻寬，**但放棄 emissive/fog**
- **Stage 3 (Subpass)**：節省 ~75% BasePass→Light 頻寬，**保留所有功能但 SceneColor 不能 strip**

→ **若你的場景對 emissive/fog 完全不需要 → 選 Stage 2**  
→ **若需要部分材質 emissive 但能接受 SceneColor 全程在 tile 中 → 選 Stage 3**

我會推薦 **Stage 3** 作為下一步，因為它向後相容、不犧牲材質能力。
